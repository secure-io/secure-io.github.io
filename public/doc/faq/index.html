What does “provable secure” mean?

TL;DR: The authenticated encryption scheme used by Secure I/O is secure if the underlying AEAD, e.g. AES-GCM, is secure.

In modern cryptography people often try to proof the security of algorithms or protocols. The advantage of a security proof is that you know - for sure - that the scheme guarantees certain security properties.

However, a security proof does neither mean that the scheme cannot be broken at all nor that it can be applied out of context. Often there is no proof showing the absolute security of a certain algorithm. Instead, many security proofs look like this statement:

If X has some property P then Y guarantees property Q.
Proof: …

In particular, the authenticated encryption scheme used by Secure I/O guarantees a set of security properties if the underlying AEAD provides confidentiality (IND-CPA) and integrity (INT-CTXT).
